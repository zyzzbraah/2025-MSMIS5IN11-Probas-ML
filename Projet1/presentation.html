LE DEFI BIOLOGIQUE

L'expression des gènes est régulée par des protéines (Facteurs de Transcription) qui se fixent sur des séquences d'ADN spécifiques : les MOTIFS.

CARACTERISTIQUES DU PROBLEME

  • Taille: Court (5-15 nucléotides) vs Génome (Milliards).
  • Variabilité: Le motif n'est jamais identique (mutations).
  • Inconnu: On ignore sa position et sa forme exacte.

CAPTURE L'INCERTITUDE (ENTROPIE)

POURQUOI C'EST DIFFICILE ?

COMPLEXITE COMBINATOIRE 

Si la position est inconnue dans N séquences de longueur L :

L^N combinaisons possibles

Pour 70 séquences de 100 nucléotides : 100^{70} possibilités. Impossible à tester par force brute.

Algo Classique (Gibbs Sampling)

Stochastique, lent, peut rester coincé dans des optima locaux.

Infer.Net (Message Passing)

Déterministe (EP/VMP), converge souvent plus vite vers une solution approximée de haute qualité.

VAR LATENTE 1 : LE MOTIF (PFM)

Quelle est la probabilité des lettres A, C, G, T à chaque position ?

DISTRIBUTION DE DIRICHLET

C'est une "distribution sur des distributions".

paramètre \alpha (Pseudocounts) :
Si \alpha est faible (< 1), on favorise des distributions "pointues" (Motif très conservé).
Si \alpha est élevé, on favorise l'uniformité (Bruit).

// Définition du Prior sur le Motif Vector pseudoCounts = Vector.Constant(4, 1.0); Range motifLen = new Range(10); // Variable tableau de vecteurs (4 dims) var motif = Variable.Array<Vector>(motifLen); motif[motifLen] = Variable.Dirichlet(pseudoCounts) .ForEach(motifLen);

VAR 2 : POSITION

Où commence le motif ?

var pos = Variable.DiscreteUniform(seqLen - motifLen);

Hypothèse : Equiprobabilité sur toute la séquence. Pas de zone privilégiée a priori.

VAR 3 : PRESENCE

Le motif est-il là ? (Modèle ZOOPS)

var isPresent = Variable.Bernoulli(0.8);

Hypothèse : On estime à 80% la chance qu'une séquence contienne le motif. Cela rend le modèle robuste aux "déchets" (Outliers).

PHASE_03 // IMPLEMENTATION

SYNTAXE : VARIABLE.FOREACH

Infer.Net utilise une syntaxe particulière pour définir des boucles sur les variables aléatoires.

Le bloc Variable.ForEach(range) ne s'exécute pas comme une boucle C# normale. Il construit le Graphe Factoriel déplié.

PLATE NOTATION :
En probabilités, cela correspond à une "Plate" (Boîte) qui répète la structure du modèle pour chaque séquence i de 1 à N.
using (Variable.ForEach(sequenceRange)) { using (Variable.If(motifPresence[sequenceRange])) { // Génération conditionnelle var motifString = Variable.StringFromArray(motifChars); // Contrainte déterministe sequences[sequenceRange] = bgLeft + motifString + bgRight; } }

LE MOTEUR D'INFERENCE

1. OBSERVED VALUE

On "cloue" les variables aléatoires sur la réalité.

sequences.ObservedValue = realData;

Le modèle ne peut plus "imaginer" les séquences, il doit maintenant expliquer celles qui existent.

2. MESSAGE PASSING (EP)

Infer.Net utilise Expectation Propagation.

  • Chaque nœud du graphe envoie des messages (distributions) à ses voisins.
  • Iteratif : On répète jusqu'à convergence.
  • Approximation : Les distributions complexes sont projetées sur des familles exponentielles (Gaussiennes, Beta, Dirichlet).

CONCLUSION

AVANTAGES INFER.NET

  • Déclaratif: On décrit le modèle, pas l'algo.
  • Incertitude: Sortie probabiliste riche.
  • Data Efficiency: Marche sur petits jeux de données.

LIMITES

  • Scalabilité: Difficile sur génome entier (L > 10^6).
  • Optima Locaux: Nécessite `Rand.Restart()`.
